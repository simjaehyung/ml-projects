## 📋 Executive Summary

v2.0 업데이트는 **기존 AABB(바운딩 박스) 기반 Object Detection에서 자유형 다각형(Polygon) 기반 Instance Segmentation 라벨링으로의 확대**와, 최신 Computer Vision 기법인 **Copy-Paste Augmentation**의 추가를 중심으로 진행되었습니다.

### 2. **backend/augmentation_recipe.py** (+280줄)

#### 신규 함수: `transform_polygon_homography`
```python
def transform_polygon_homography(
    points: List[List[float]],  # [[x1,y1], [x2,y2], ...] 
    H: np.ndarray,              # 3×3 Homography 행렬
    img_w: int, img_h: int      # 이미지 경계 (클리핑용)
) -> List[List[int]]:
```

**핵심 수학:**
```
Input:  N개 점 배열
Step 1: 동차 좌표 변환 → (3, N) 행렬
        [[x1..xN],
         [y1..yN],
         [1..1  ]]
         
Step 2: H @ pts_h → (3, N) 변환된 동차 좌표

Step 3: Perspective Divide (원근 나눗셈)
        xs = transformed[0] / W
        ys = transformed[1] / W
        (W가 0에 가까우면 1e-10으로 대체)
        
Step 4: 이미지 경계 클리핑
        xs = np.clip(xs, 0, img_w)
        ys = np.clip(ys, 0, img_h)
        
Output: [[x'1,y'1], [x'2,y'2], ...] 정수 좌표
```

**설계 이유:**
- AABB(4점)만 변환하면 원근 후 사각형으로 모양 손실
- Polygon(N점 전체)을 변환하면 객체 실루엣 정확히 유지
- Instance Segmentation에 필수

#### 신규 함수: `apply_copy_paste`
```python
def apply_copy_paste(
    primary_image: np.ndarray,
    primary_annotations: List[dict],
    background_pool: List[np.ndarray],
    num_pastes: int = 1
) -> Tuple[np.ndarray, List[dict]]:
```

**5단계 파이프라인:**

| 단계 | 동작 | 코드 |
|------|------|------|
| 1 | Polygon → 이진 마스크 | `cv2.fillPoly(mask, poly_pts, 255)` |
| 2 | 마스크 기반 누끼 추출 | `cv2.bitwise_and(image, mask_3ch)` |
| 3 | 배경 선택 + 크기 맞춤 | `cv2.resize(bg_raw, (img_w, img_h))` |
| 4 | 객체 합성 | `result_image[dy:dy+oh, dx:dx+ow] = blend(...)` |
| 5 | 좌표 동기화 | `ann['points'] = [[px+shift_x, py+shift_y], ...]` |

**핵심 특징:**
- 배경 픽셀 오염 방지 (마스크 기반)
- 투명도 100% (배경 완전 교체)
- 합성 후 polygon 좌표 평행 이동 (라벨 정확히 동기화)

#### `__getitem__` 수정 (Homography 분기)
```python
if ann.get('type') == 'polygon' and ann.get('points'):
    # Polygon N개 점 변환
    ann['points'] = transform_polygon_homography(
        ann['points'], H, img_w, img_h

else:
    # 기존 AABB 4점 변환
```

## 🔬 기술적 깊이

### 1. Polygon Homography의 수학적 정당성

**문제:** 직사각형(AABB)에 원근 변환을 적용하면 일반 사각형(Quadrilateral)이 됨

**해결:** 경계가 아닌 **모든 N개 점**을 벡터화로 변환

```
동차 좌표계(Homogeneous Coordinates):
  2D 점 (x, y) → 3D 벡터 [x, y, 1]^T

Homography 행렬 적용:
  [x']     [h11 h12 h13] [x]
  [y']  =  [h21 h22 h23] [y]
  [w]      [h31 h32 h33] [1]

Perspective Divide (유클리드 좌표 복원):
  x_final = x' / w
  y_final = y' / w
  
이 과정이 원근감(perspective)을 만듦
```

**성능:** O(N) 벡터화 연산 (NumPy)

---

### 2. Copy-Paste의 Instance Segmentation 영향

**기존 문제 (Mosaic 등):**
- 단순 이미지 자르기만 → 배경 픽셀 포함
- 라벨과 실제 객체 영역 불일치

**Copy-Paste 해결:**
- Polygon 마스크로 **정확한 객체 경계** 추출
- 배경은 완전 교체 (노이즈 제거)
- Instance Segmentation 학습에 최적화

**식:**
```
누끼_이미지 = 원본_이미지 ⊗ 이진_마스크  (⊗: bitwise AND)
합성_이미지 = 배경 ⊗ (¬마스크) + 누끼 ⊗ 마스크
라벨_이동 = 원본_라벨 + (dx - obj_x, dy - obj_y)
```
