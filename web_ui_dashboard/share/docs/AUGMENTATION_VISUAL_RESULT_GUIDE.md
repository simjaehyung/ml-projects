## 🔴 발견된 버그: "강한 빛 찢어짐 선 + 대칭"

### 증상 (사용자 관측)
```
[원본]          [증강 후 이상 현상]

┌────────┐      ┌────────┬────────┐
│        │      │  이미지 │거울반사│
│  사진  │  →   │        │  사진  │
│        │      │        │        │
└────────┘      └────────┴────────┘
                         ↑
               강한 빛 선처럼 보이는 "경계선"
```

### 원인 분석

**파일:** `app/backend/augmentation_recipe.py` — `apply_homography()` 함수

```python
# ❌ 버그 코드 (수정 전)
warped = cv2.warpPerspective(
    image, H, (w, h),
    borderMode=cv2.BORDER_REFLECT_101  ← 이게 문제!
)
```

**`BORDER_REFLECT_101` 동작 원리:**
```
원본 픽셀:  A B C D E
경계 밖:    D C B | A B C D E | D C B A
                ↑
         여기서 이미지가 거울처럼 반전!
         이 경계가 밝고 선명한 수직/수평 선으로 보임
```

원근 변환 후 이미지 일부가 경계 밖으로 밀려나면, 그 빈 공간을 **이미지를 거울처럼 뒤집어서** 채웁니다. 이게 사용자 눈에:
- 이미지 중간에 **대칭 선**(찢어짐처럼 보임)
- 갑자기 방향이 바뀌는 **밝은 경계선**

으로 보입니다.

### 수정 코드

```python
# ✅ 수정 후
warped = cv2.warpPerspective(
    image, H, (w, h),
    borderMode=cv2.BORDER_REPLICATE  ← 테두리 픽셀 복사
)
```

**`BORDER_REPLICATE` 동작 원리:**
```
원본 픽셀:  A B C D E
경계 밖:    A A A | A B C D E | E E E E
                ↑
         가장자리 색을 그냥 쭉 늘림 (자연스러운 배경)
```

### 수정 전 vs 수정 후 비교

| 항목 | 수정 전 (REFLECT_101) | 수정 후 (REPLICATE) |
|------|----------------------|-------------------|
| 경계 처리 | 이미지 거울 반사 | 테두리 색상 확장 |
| 시각 효과 | 대칭 선 + 찢어짐 | 자연스러운 확장 |
| 객체 위치 | BBox 정확 | BBox 정확 |
| 현실감 | 낮음 (비물리적) | 높음 (실제 카메라처럼) |

### 1️⃣ Homography (원근 + 회전)

**기대되는 시각 결과:**

```
원본:                  증강 후:
┌──────────────┐      ┌──────────────┐
│   🚗 자동차   │      │  🚗자동차    │ ← 약간 기울어짐
│              │  →   │ (사다리꼴처럼) │
└──────────────┘      └──────────────┘
                       (±5° 이내의 미세한 변화)
```

**정상 범위:**
- 회전: **최대 ±5°** (거의 티 안 날 수준)
- 원근: **매우 미세** (h31, h32 = ±0.001)
- 결과: 이미지가 아주 살짝 기울거나 사다리꼴처럼 보임

**수정 전 오류 증상:**
- 이미지 한가운데에 수직/수평으로 선명한 선
- 선 너머로 이미지가 좌우 반전
- 마치 이미지를 반으로 접어 붙인 것처럼 보임

**수정 후 정상 모습:**
- 이미지 가장자리가 배경색(테두리 픽셀)으로 자연스럽게 채워짐
- 뚜렷한 선이나 반전 없음
- 실제 카메라 시점이 살짝 달라진 것처럼 보임

**BBox 변화:**
- 원본 박스보다 약간 기울어진 위치로 이동
- 박스 크기는 비슷하게 유지

---

### 2️⃣ Poisson Noise (카메라 센서 노이즈)

**기대되는 시각 결과:**

```
원본:                  증강 후:
┌──────────────┐      ┌──────────────┐
│  선명한 사진  │      │ 약간 거친 사진│ ← 미세한 입자감
│              │  →   │ (TV 노이즈처럼│
└──────────────┘      │  아주 약하게) │
                       └──────────────┘
```

**정상 범위:**
- λ 값: 20~40
- 노이즈 강도: **밝은 영역은 강하게, 어두운 영역은 약하게**
- 전체적으로 사진에 미세한 "필름 입자감" 느낌

**잘못된 경우:**
- λ가 너무 작으면: 이미지 전체가 어두워짐
- 큰 λ는 오히려 Gaussian 노이즈와 비슷해짐 (CLT 수렴)

**BBox 변화:** 없음

---

### 3️⃣ Gamma Correction (밝기 조정)

**기대되는 시각 결과:**

```
γ < 1.0 (예: 0.5):         γ > 1.0 (예: 2.0):
┌──────────────┐            ┌──────────────┐
│  더 밝아짐   │            │  더 어두워짐  │
│  디테일 살아남│            │  과노출 보정  │
└──────────────┘            └──────────────┘

           γ = 1.0 (원본 유지)
```

**정상 범위:**
- γ 범위: 0.6 ~ 1.6 (기본값, UI에서 변경 가능)
- 색조는 유지되고 **밝기만 균일하게 변화**
- γ = 0.8: 살짝 밝아짐 (야간 이미지 처리처럼)
- γ = 1.4: 살짝 어두워짐 (과노출 보정처럼)

**BBox 변화:** 없음

---

### 4️⃣ CLAHE (히스토그램 명암비)

**기대되는 시각 결과:**

```
원본 (명암 대비 낮음):       증강 후 (명암비 향상):
┌──────────────┐            ┌──────────────┐
│ 전체적으로   │            │ 밝은 부분은 │
│ 뿌옇게       │      →     │ 더 밝고,    │
│ 보이는 이미지│            │ 어두운 부분은│
└──────────────┘            │ 더 어두워짐  │
                             └──────────────┘
                             (색상은 변하지 않음!)
```

**정상 범위:**
- Clip Limit: 1.0 (약함) ~ 5.0 (강함), 기본 2.0
- **중요:** 색상(Red, Blue, Green의 비율)은 변하지 않음
- 오직 **밝기(Y 채널)** 만 처리

**주의:**
- Clip Limit이 너무 높으면 노이즈가 심해질 수 있음
- YCrCb 방식을 쓰기 때문에 색조 왜곡은 없음

**BBox 변화:** 없음

---

### 5️⃣ Cutout (Random Erasing)

**기대되는 시각 결과:**

```
원본:                  증강 후:
┌──────────────┐      ┌──────────────┐
│   🚗 자동차   │      │ 🚗 자동차    │
│              │  →   │     ████     │ ← 회색/검정 사각형
└──────────────┘      └──────────────┘
                            ↑
                    이미지 면적의 2%~10%
                    (fill_mode='mean': 주변 평균색)
```

**정상 범위:**
- 크기: 전체 이미지의 **2% ~ 10%** 면적 (UI에서 최대치 설정)
- 종횡비: 0.3 ~ 3.0 범위 (가로나 세로로 긴 직사각형도 가능)
- 위치: 완전히 랜덤 (이미지 어디든)
- 색상: 평균 픽셀값(회색 계열) 또는 검정

**BBox 변화:**
- 기본: 유지 (Cutout이 박스를 가려도 박스 정보 그대로)
- drop_covered=True: Cutout이 BBox를 **100% 완전히 덮으면** 해당 BBox 제거

### 6️⃣ Copy-Paste (객체 누끼 합성)

> **주의: Polygon 라벨이 있는 경우에만 작동!**

**기대되는 시각 결과:**

```
원본 (Polygon 라벨):    배경 이미지:         합성 결과:
┌──────────────┐        ┌──────────────┐     ┌──────────────┐
│  /🚗\        │        │              │     │              │
│ /    \       │  +     │  도로 사진    │  →  │   도로 위에  │
│ Polygon      │        │              │     │  🚗이동된 위치│
└──────────────┘        └──────────────┘     └──────────────┘
                                                      ↑
                                            Polygon 좌표도 이동됨
```

**정상 동작:**
1. Polygon 경계선만 정확히 오려냄 (배경 픽셀 없음)
2. 무작위 배경 이미지 선택
3. 무작위 위치에 붙여넣음
4. Polygon 라벨 좌표 자동 이동

**이 기법이 없을 때 vs 있을 때:**

```
Copy-Paste 없음:              Copy-Paste 있음:
┌──────────────┐              ┌──────────────┐
│ 자동차+배경  │              │ 자동차만     │
│ (배경 오염)  │              │ (누끼 정확)  │
└──────────────┘              └──────────────┘
```

## 🖼️ 전체 파이프라인 적용 순서

```
원본 이미지 (1장)
      │
      ▼
 【1단계: Homography】
   이미지가 살짝 기울고 원근감 생김
   BBox도 함께 변환 (AABB 재계산)
      │
      ▼
 【2단계: Poisson Noise】
   전체에 미세한 입자 노이즈 추가
   BBox 변화 없음
      │
      ▼
 【3단계: Gamma Correction】
   전체 밝기 균일하게 조정
   BBox 변화 없음
      │
      ▼
 【4단계: CLAHE】
   명암비 국소 강화
   BBox 변화 없음
      │
      ▼
 【5단계: Cutout】
   이미지 일부 회색/검정으로 가림
   BBox 유지 (완전 가림 시 드롭)
      │
      ▼
 【6단계: Copy-Paste】
   Polygon 라벨 객체를 다른 배경에 합성
   Polygon 좌표 이동
      │
      ▼
 증강 이미지 완성 + 오버레이 텍스트 추가
 → project_data/images/augmented/에 저장
```

## 🏷️ 신규 기능: 적용 기법 오버레이

증강 이미지 저장 시 우하단에 어떤 기법이 적용됐는지 표시됩니다.

**표시 형식 예시:**

```
┌────────────────────────────────────────────┐
│                                            │
│            증강된 이미지                    │
│                                            │
│                    ┌───────────────────┐   │
│                    │ Homography th=-2° │   │
│                    │ Poisson    lam=28 │   │
│                    │ Gamma      g=0.74 │   │
│                    │ CLAHE      clip=2 │   │
│                    │ Cutout     8%     │   │
│                    │ Copy-Paste        │   │
│                    └───────────────────┘   │
└────────────────────────────────────────────┘
```

**구현 위치:**
```
app/app.py: draw_augmentation_labels(image, params)
            → /api/save 라우트의 저장 루프에서 호출
```

**파라미터 표시 기준:**

| 기법 | 표시 조건 | 표시 내용 |
|------|---------|---------|
| Homography | `theta_deg` 또는 `h31` 있을 때 | 회전각 표시 |
| Poisson | `poisson_lambda` 있을 때 | λ 값 표시 |
| Specular Flare | `specular_flare: True` 일 때 | 이름만 |
| Shadow | `shadow: True` 일 때 | 이름만 |
| Gamma | `gamma` 있을 때 | γ 값 표시 |
| CLAHE | `clahe_clip_limit` 있을 때 | clip 값 표시 |
| Cutout | `cutout_area_max` 있을 때 | 면적 % 표시 |
| Mosaic | `mosaic: True` 일 때 | 이름만 |
| Copy-Paste | `copy_paste: True` 일 때 | 이름만 |

## 📊 "이상하다" vs "정상이다" 구분 기준

### ✅ 정상적인 증강 이미지의 특징

| 체크 | 정상 | 이상 |
|------|------|------|
| Homography | 이미지가 살짝 기울어짐 | 이미지 중간에 선이 생기거나 절반이 반전됨 |
| Poisson | 전체에 미세한 필름 입자감 | 색상 패턴이 바뀌거나 전체가 밝아짐/어두워짐 |
| Gamma | 전체 밝기가 균일하게 변화 | 특정 색상만 변하거나 무지개 효과 |
| CLAHE | 명암 대비 향상, 색상 유지 | 녹색/보라 등 색 왜곡 |
| Cutout | 이미지 어딘가에 회색 박스 | 박스가 전체를 덮음 |
| Copy-Paste | 객체가 다른 위치에 자연스럽게 합성 | 배경 포함된 채 복사됨 |

### ❌ 수정 전 버그 체크리스트

- [x] **"강한 빛 찢어짐 선"** → BORDER_REFLECT_101에 의한 거울 반사 경계선 → **수정 완료**
- [x] **"대칭 이미지"** → 원근 변환 후 이미지 일부가 거울처럼 복사됨 → **수정 완료**
