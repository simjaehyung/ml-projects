## 2. 구현 상세

### 2.1 Gamma Correction (`apply_gamma_correction`)

**수식:**
$$V_{out} = 255 \times \left(\frac{V_{in}}{255}\right)^{\gamma}$$

**구현 핵심 — LUT(Look-Up Table) 최적화:**
```python
lut = np.array([np.clip(255.0 * (i / 255.0) ** gamma, 0, 255) for i in range(256)], dtype=np.uint8)
result = cv2.LUT(image, lut)
```

왜 LUT를 쓰는가:
- **브루트 포스**: 이미지의 모든 픽셀(H×W×C개)에 지수 연산(고비용) 적용 → O(HWC)
- **LUT 방식**: 0~255의 256개 정수에만 연산 → O(256) 사전 계산 후 모든 픽셀에 O(1) 테이블 참조
- 1920×1080 3채널 이미지 기준 약 622만 번의 지수 연산을 256번으로 줄임 → **약 24,000× 절감**

**γ 값의 물리적 의미:**
- `γ < 1` (예: 0.5): 어두운 영역 밝아짐 → 야간 촬영, 언더노출 시뮬레이션
- `γ > 1` (예: 2.0): 전체 밝기 감소 → 강한 광원 아래 촬영 시뮬레이션
- `γ = 1`: 항등 변환 (원본 그대로)

**BBox 처리:** 공간 변환 없음 → 좌표 유지

---

### 2.2 CLAHE 히스토그램 평탄화 (`apply_histogram_equalization`)

**왜 BGR 직접 처리가 아닌 YCrCb인가:**

BGR 각 채널을 독립 처리하면 채널 간 밝기 비율이 깨져 색조가 왜곡된다.  
YCrCb 색공간은 밝기(Y)와 색차(Cr, Cb)를 분리하므로 Y만 처리하면 색상이 완전히 보존된다.

```
BGR → YCrCb → Y 채널에만 CLAHE → YCrCb 합성 → BGR
```

**일반 equalizeHist vs CLAHE:**

| 방식 | 처리 단위 | 노이즈 증폭 | 적합 상황 |
|------|-----------|------------|-----------|
| `equalizeHist` | 전역 | 높음 | 단순 저대비 이미지 |
| CLAHE | 타일(8×8 등) | clip limit으로 제한 | 실제 영상 처리 |

`clipLimit=2.0`, `tileGridSize=(8,8)`이 CLAHE의 기본값으로, 대부분의 Object Detection 데이터에 적합하다.

**BBox 처리:** 공간 변환 없음 → 좌표 유지

---

### 2.3 Cutout / Random Erasing (`apply_cutout`)

**알고리즘 단계:**

1. **면적 결정:** $S_{cut} = S_{img} \times \text{Uniform}(r_{min}, r_{max})$
2. **종횡비 결정:** $r = \text{Uniform}(0.3, 3.0)$
3. **박스 크기 계산:**
   $$cut_h = \sqrt{S_{cut} / r}, \quad cut_w = \sqrt{S_{cut} \times r}$$
4. **위치 샘플링:** $(rx, ry)$ 무작위 선택 + 이미지 경계 클리핑
5. **픽셀 채우기:** NumPy 슬라이싱으로 O(1)에 처리

**fill_mode 선택:**
- `'black'`: 0으로 채움 (극단적 오클루전 모사)
- `'mean'`: 채널별 평균값으로 채움 (배경과 유사한 자연스러운 처리)

**BBox drop 로직 (`drop_covered=True`):**
```
Cutout이 BBox를 완전히 포함 조건:
  rx ≤ bx  AND  ry ≤ by  AND  rx+cw ≥ bx+bw  AND  ry+ch ≥ by+bh
```
이 조건을 만족하는 BBox만 어노테이션 목록에서 제거한다.  
IoU 계산이 아닌 완전 포함 여부(Containment Check)를 사용하여 연산을 단순화했다.

### 2.4 Mosaic Augmentation (`apply_mosaic`)

YOLOv4에서 처음 도입된 기법. 4개 이미지를 단일 캔버스에 배치해 다음을 달성한다:
- 다양한 배경/컨텍스트 동시 학습
- 소형 객체에 대한 감지 성능 향상
- 배치 정규화(Batch Normalization)가 더 다양한 통계를 학습

**수학적 변환 파이프라인:**

```
1. 교차점 결정:
   cx = Uniform(W×0.3, W×0.7)
   cy = Uniform(H×0.3, H×0.7)

2. 4개 사분면 정의:
   Q0 (TL): canvas[0:cy,   0:cx  ]  ← primary image
   Q1 (TR): canvas[0:cy,   cx:W  ]
   Q2 (BL): canvas[cy:H,   0:cx  ]
   Q3 (BR): canvas[cy:H,   cx:W  ]

3. 각 사분면 i에 대해:
   src 이미지를 (quad_w, quad_h)로 리사이즈:
     scale_x = quad_w / src_w
     scale_y = quad_h / src_h

4. BBox 좌표 변환:
   new_x = bbox.x × scale_x + x_offset
   new_y = bbox.y × scale_y + y_offset
   new_w = bbox.w × scale_x
   new_h = bbox.h × scale_y

5. AABB 클리핑 (캔버스 경계 초과 방지):
   new_x  = clip(new_x,       0, output_w)
   new_y  = clip(new_y,       0, output_h)
   new_x2 = clip(new_x+new_w, 0, output_w)
   new_y2 = clip(new_y+new_h, 0, output_h)
   → 유효 박스(w≥1, h≥1)만 유지
```

**이미지 풀 로딩 전략:**
- `project_data/images/original/` 폴더에서 무작위로 이미지를 로드
- `dataset_index.json`이 존재하면 해당 이미지의 어노테이션도 함께 로드
- 이미지가 3장 미만이면 primary image로 패딩 (항상 4장 보장)

## 3. 파이프라인 연동 (app.py)

### 3.1 신규 파라미터 목록

`run_augmentation_pipeline` 함수에 추가된 파라미터:

| 파라미터 | 타입 | 기본값 | 설명 |
|----------|------|--------|------|
| `enable_gamma` | bool | False | Gamma Correction 활성화 |
| `gamma_range` | tuple | (0.6, 1.6) | γ 무작위 샘플링 범위 |
| `enable_clahe` | bool | False | CLAHE 활성화 |
| `clahe_clip_limit` | float | 2.0 | CLAHE 명암비 증폭 한계 |
| `enable_cutout` | bool | False | Cutout 활성화 |
| `cutout_area_max` | float | 0.08 | Cutout 최대 면적 비율 (8%) |
| `enable_mosaic` | bool | False | Mosaic 활성화 |
| `original_dir` | str | None | Mosaic 이미지 풀 경로 |
| `source_filename` | str | '' | Mosaic 시 자기 자신 제외용 |

### 3.2 증강 적용 순서 (중요)

```
[입력 이미지]
    ↓
1. Homography (기하학적 변환 + BBox 동기화)
    ↓
2. Poisson Noise (밝기 의존 노이즈)
    ↓
3. Specular Flare / Shadow (조명 효과)
    ↓
4. Gamma Correction (비선형 밝기 조정)
    ↓
5. CLAHE (히스토그램 평탄화)
    ↓
6. Cutout (오클루전 + BBox 드롭)
    ↓
7. Mosaic (4-이미지 합성 + BBox 좌표 재계산)
    ↓
[최종 좌표 클램핑 + 정수 변환]
    ↓
[출력: (증강 이미지, 어노테이션, 파라미터 딕셔너리)]
```

왜 이 순서인가:
- **기하학(1) → 노이즈(2-3) → 밝기(4-5)**: 기하 변환 후 센서 특성 시뮬레이션이 자연스러운 순서
- **Cutout(6) before Mosaic(7)**: Cutout은 단일 이미지 처리이므로 Mosaic(4개 이미지 합산) 이전에 적용
- **Mosaic 마지막**: 여러 이미지를 합친 후 다시 다른 변환을 가하면 왜곡이 중첩되어 비현실적

## 6. 리스크 및 예외 처리

### 6.1 구현된 방어 로직

```python
# app.py - 각 신규 증강을 개별 try-except로 감싸 파이프라인 중단 방지
if enable_gamma:
    try:
        aug_img = apply_gamma_correction(aug_img, gamma_val)
    except Exception as e:
        logging.warning(f"Gamma Correction 실패: {e}")
        # 증강 실패 시 해당 단계 건너뜀, 이미지는 이전 단계 결과 유지
```

### 6.2 알려진 제한사항

| 상황 | 현재 처리 | 향후 개선 방향 |
|------|-----------|---------------|
| Mosaic: 이미지 풀 0장 | 조건 `len(mosaic_pool) >= 3` 미충족 → 스킵 | 경고 메시지를 API 응답에 포함 |
| Cutout area > bbox area | drop 안 됨 (완전 포함이 아니므로) | partial IoU threshold 옵션 추가 |
| CLAHE: 그레이스케일 입력 | YCrCb 변환 실패 가능 | shape 체크 후 grayscale 분기 |

### 7.3 Python 코드에서 직접 호출
```python
from app.backend.augmentation_recipe import (
    apply_gamma_correction,
    apply_histogram_equalization,
    apply_cutout,
    apply_mosaic,
)
import cv2

image = cv2.imread('sample.jpg')
anns = [{'label': 'car', 'x': 100, 'y': 50, 'w': 200, 'h': 100}]

# Gamma: 어두운 이미지 밝히기 (γ=0.6)
bright = apply_gamma_correction(image, gamma=0.6)

# CLAHE: 대비 향상
enhanced = apply_histogram_equalization(image, use_clahe=True, clip_limit=2.0)

# Cutout: 8% 면적 마스킹
masked, new_anns = apply_cutout(image, anns, area_ratio_max=0.08, drop_covered=True)

# Mosaic: 3개 추가 이미지와 합성
pool = [(cv2.imread(f'img{i}.jpg'), []) for i in range(3)]
mosaic_img, mosaic_anns = apply_mosaic(image, anns, pool, output_w=640, output_h=480)
```
