이미지 증강 프롬프트
[우리의 역할과 목표] 당신은 1조의 'Object Detection을 위한 데이터 라벨링 및 증강' UI 데모를 구축하는 프론트엔드/백엔드 AI 엔지니어입니다. 핵심 목표는 사용자가 스크랩한 ROI(이미지)를 기반으로, 현장에서 수용 가능한 오차 범위(Tolerance)를 설정하고 그 범위 내에서 랜덤 샘플링된 대량의 데이터를 On-the-fly로 안전하게 증강하는 시스템을 구현하는 것입니다.

[구현해야 할 기능 및 안전 설계 상세]

데이터 유효 범위(Tolerance) 제어 기반 증강 UI:
$3 \times 3$ 호모그래피 투영 행렬($h_{31}, h_{32}$)과 푸아송 노이즈 센서 감도($\lambda$)의 최대 허용 한계치를 사용자가 슬라이더로 조절하게 하세요.
너무 과도한 변환은 학습을 해치므로, 시스템이 제안하는 기본 '디폴트(적정 변환 척도)'를 제공하세요.
슬라이더 조작 시, 최대로 변환/노이즈가 적용된 한계점 이미지를 캔버스에서 미리보기로 보여주세요.
증강 실행 시, 정상 상태부터 사용자가 설정한 한계치 사이에서 무작위로 파라미터를 추출하여, 사용자가 지정한 '생성 개수'만큼 다량의 증강 세트를 생성하는 파이프라인 뼈대를 짜세요.
라벨 보존(Label Preserving) 철저 방어:
방향성이 중요한 객체('6', '9', 화살표 등)가 증강 시 의미를 잃지 않도록, 클래스 메타데이터를 기반으로 상하반전(Flip)이나 90도 이상의 큰 회전 등 특정 증강 로직을 자동 비활성화하는 예외 처리 필터를 파이프라인에 반드시 넣으세요.
바운딩 박스(ROI) 연산 정밀화 (Min-Max 투영):
호모그래피 변환 적용 시 좌표가 망가지는 것을 막아야 합니다. 원본 ROI 바운딩 박스의 [Left, Top, Right, Bottom]을 토대로 4개의 꼭짓점 좌표를 구한 뒤, 동일한 $H$ 투영 행렬을 곱하세요.
변환된 4개의 새로운 꼭짓점에서 min(X), max(X), min(Y), max(Y)를 다시 도출해 항상 화면에 평행한 올바른 직사각형(Axis-aligned Bounding Box) 라벨로 갱신하는 엄밀한 수학적 함수 로직을 작성하세요.
실전 조명 증강 추가:
공장 카메라 환경에 맞춰, 단순 어두운 노이즈(Poisson)뿐만 아니라 금속 표면의 국소적인 빛 반사 덩어리(Specular Flares)나 그라데이션 그림자 영역을 추가로 합성하는 증강 필터 옵션을 구현하세요.
효율적 백엔드 저장 구조 및 On-the-fly 지원:
수백 장으로 부풀린 이미지를 매번 물리적 파일(.jpg, .png 등)로 무식하게 저장하여 서버 용량을 낭비하지 마세요.
원본 이미지 데이터의 UUID, 바운딩 박스 좌표, 그리고 여기에 적용할 증강 설정값 파라미터 묶음(예: 회전각, h31, lambda 등)만 가볍게 JSON 리스트 형식으로 Backend DB에 저장하는 스키마를 구성하세요.
차후 이 JSON 스키마를 읽어들여 모델이 학습 배치를 구성할 때 메모리단에서 실시간으로 이미지를 증강(On-the-fly)할 수 있도록 구조를 설계하세요.
