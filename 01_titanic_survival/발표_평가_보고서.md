# 프로젝트 발표 평가 및 코멘트 보고서

> **안내**: 제공해주신 텍스트에는 **1조의 발표 내용과 피드백**만 포함되어 있었습니다. 요청하신 대로 총 6개 조에 대해 코멘트를 쉽게 달 수 있도록 보고서 양식을 구성하였으며, 1조의 내용은 스크립트를 바탕으로 요약해 채워두었습니다. 2조부터 6조까지는 추후 내용을 기록하실 수 있도록 빈 양식으로 준비했습니다.

## 1조
**주제:** 오브젝트 디텍션을 위한 데이터 라벨링 및 증강 구현 파이프라인

### 1. 발표 내용 요약
* **프로젝트 개요**
  * 오브젝트 디텍션 모델 학습 시 라벨링된 이미지 데이터가 부족한 문제를 해결하기 위함.
  * 웹 기반 라벨링 시스템 및 데이터 증강 파이프라인 구축.
* **시스템 구조**
  * **프론트엔드 (웹 UI)**: 캔버스에 바운딩 박스/폴리곤을 그리고 라벨 입력. 히스토리 스냅샷으로 실행 취소(Ctrl+Z) 지원.
  * **백엔드**: 입력된 데이터 검증 및 처리.
  * **스토리지**: JSON 기반으로 데이터 및 데이터셋 인덱스를 파일 시스템에 저장/관리.
* **주요 증강 기법**
  1. **호모그래피 (Homography)**: 이미지의 각도와 원근법을 변환하여 다양한 시점을 학습.
  2. **포아송 노이즈 (Poisson Noise)**: 빛이 적은 환경에서 발생하는 노이즈를 인위적으로 생성.
  3. **라이트닝 (Lightning)**: 밝기 및 그림자 등 조명 변화 추가.
  4. **랜덤 이레이징 (Random Erasing)**: 객체의 일부분이 가려진 상황(Occlusion)을 구현.
  5. **모자이크 (Mosaic)**: 여러 이미지를 합성하여 다양한 배경과 다중 객체가 등장하는 상황 학습.
* **결론 및 성과**
  * 웹 UI에서 이미지 업로드 → 객체 라벨링 → 데이터 저장 → 데이터 증강 → 학습 데이터셋 생성(COCO, YOLO 형식 Export 지원)까지의 전체 파이프라인 구현 완료.
* **아쉬운 점 (한계 및 개선 방향)**
  * **이미지 경계 검증**: 이미지 밖에서 바운딩 박스가 생성되는 문제 미해결. (향후 마우스 이벤트 단계에서 좌표 제한으로 개선 예정)
  * **폴리곤 라벨 편집**: 점 이동/추가/삭제 기능 부재. (향후 드래그 기반 편집 기능 추가 예정)

### 2. 질의응답(Q&A) 및 교수님 피드백
* **아쉬운 점 발표 방식 개선**: 해당 기능이 왜 필요한지, 없으면 왜 문제가 되는지를 먼저 설명하고, 그에 따른 구체적인 개선 및 추가 계획을 이야기하는 것이 설득력이 높음.
* **기술적 질문 대비**: 아쉬운 점으로 언급한 부분(예: 이미지 경계 검증)에 대해 "어떻게 해결할 것인가?"라는 질문이 들어왔을 때 당황하지 않도록 아이디어(예: 좌표가 최대값을 넘으면 최대값으로 고정 등)를 미리 생각해 두어야 함. 수학적이거나 복잡한 내용이더라도 방어 로직에 대한 고민이 필요함.
